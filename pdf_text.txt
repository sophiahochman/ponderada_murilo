 Sistema de Medição de Estação Meteorológica IoT Atividade Ponderada  
1. Apresentação Esta atividade integra os conhecimentos adquiridos ao longo do módulo sobre programação em Python, desenvolvimento de APIs REST com Flask, persistência de dados com SQLite e programação de dispositivos físicos com Arduino/ESP32. O objetivo central é construir um sistema completo de ponta a ponta: um dispositivo físico simulando uma estação meteorológica envia dados para um servidor web, que os armazena em banco de dados e os disponibiliza em uma interface de visualização. Ao concluir esta atividade, o estudante terá desenvolvido um sistema real de IoT (Internet das Coisas), com comunicação serial, API REST, banco de dados relacional e interface web responsiva. Um ponto importante que deve ser observado: o sistema possui diversos níveis para ser considerado. O que isso significa? Deve existir uma decisão de arquitetura inicial para estes elementos.

Visão Geral da Arquitetura O sistema é composto por três camadas que se comunicam entre si:  Camada Tecnologia Responsabilidade Dispositivo (Hardware) Arduino Uno + Sensores DHT11 / BMP180 Leitura e envio de dados via Serial USB Servidor (Backend) Python + Flask + SQLite Receber, armazenar e expor dados via API REST Interface (Frontend) HTML + CSS + JavaScript / Jinja2 Exibir, filtrar e gerenciar leituras

5. Parte 1 — Programação do Arduino 5.1 Circuito Monte o circuito conforme as ligações abaixo. Em caso de dúvida, consulte o professor ou o datasheet do sensor: • DHT11 VCC → 5V do Arduino • DHT11 GND → GND do Arduino • DHT11 DATA → Pino Digital 2 do Arduino (com resistor pull-up de 10 kΩ para o 5V) IMPORTANTE: é uma sugestão, pode ser alterado de acordo com o que for necessário. 5.2 Sketch base (estacao.ino) O Arduino deve enviar dados pela porta serial no formato JSON, uma linha por leitura, a cada 5 segundos. Use o código abaixo como ponto de partida:  #include "DHT.h" #define DHTPIN 2 #define DHTTYPE DHT11 DHT dht(DHTPIN, DHTTYPE);  void setup() {   Serial.begin(9600);   dht.begin(); }  void loop() {   float temp = dht.readTemperature();   float umid = dht.readHumidity();   if (!isnan(temp) && !isnan(umid)) {     Serial.print("{");     Serial.print("\"temperatura\":"); Serial.print(temp);     Serial.print(",\"umidade\":"); Serial.print(umid);     Serial.println("}");   }   delay(5000); }  
⚠  Caso os sensores físicos não estejam disponíveis, implemente um modo de simulação no próprio Arduino (ou em Python) que gere valores aleatórios realistas. Documente essa decisão no README.

6. Parte 2 — Banco de Dados SQLite 6.1 Esquema obrigatório O banco deve conter ao menos a tabela leituras com os campos abaixo. Campos adicionais são permitidos e incentivados:  CREATE TABLE IF NOT EXISTS leituras (     id          INTEGER PRIMARY KEY AUTOINCREMENT,     temperatura REAL    NOT NULL,     umidade     REAL    NOT NULL,     pressao     REAL,               -- opcional     localizacao TEXT DEFAULT 'Lab', -- opcional     timestamp   DATETIME DEFAULT (datetime('now','localtime')) ); ATENÇÃO: denovo essa é uma sugestão, pode alterar. 6.2 Módulo database.py Implemente as funções abaixo. Cada função deve ser testável de forma independente: • get_db_connection() — retorna uma conexão configurada com row_factory • init_db() — cria as tabelas se não existirem (executa o schema.sql) • inserir_leitura(temperatura, umidade, pressao=None) — INSERT • listar_leituras(limite=50) — SELECT com paginação básica • buscar_leitura(id) — SELECT por id • atualizar_leitura(id, dados) — UPDATE • deletar_leitura(id) — DELETE  6.3 IMPORTANTE: Para escrever no banco de dados Pessoal, a escrita no banco de dados por dois processos simultâneos não é possível no SQLite. Como vamos ter o Flask e a leitura Serial escrevendo no banco (se vocês optarem por utilizar a leitura serial), vai ser necessário fazer um pequeno ajuste na função de conexão com o banco. import sqlite3 def get_db_connection():        conn = sqlite3.connect('dados.db', timeout=10)        conn.execute('PRAGMA journal_mode=WAL')        conn.execute('PRAGMA busy_timeout=5000')  # espera até 5s         conn.row_factory = sqlite3.Row        return conn Como funciona: escritas vão para um arquivo de log separado (.db-wal) e são consolidadas depois. Dois processos podem escrever, mas um espera o outro terminar — sem travar a aplicação inteira.

8. Parte 4 — Leitura da Porta Serial 8.1 Script serial_reader.py Este script roda em paralelo com o Flask (um processo independente, em outro terminal) e fica continuamente lendo a porta serial do Arduino, fazendo POST para a própria API:  import serial, json, requests, time  PORTA = 'COM3'          # Windows: COM3 / Linux: /dev/ttyUSB0 BAUD  = 9600 URL   = 'http://localhost:5000/leituras'  def ler_serial():     with serial.Serial(PORTA, BAUD, timeout=2) as ser:         while True:             linha = ser.readline().decode('utf-8').strip()             if linha:                 try:                     dados = json.loads(linha)                     requests.post(URL, json=dados)                     print(f'Enviado: {dados}')                 except json.JSONDecodeError:                     print(f'Linha inválida: {linha}')             time.sleep(0.1)  if __name__ == '__main__':     ler_serial()
